emit (em, event) Emits an event calling all the callbacks registered.
enqueue (em, event) Emits an event but wraps the emission in cqueues:wrap.

__pairs (em[, timeout])
pairs iterator, iterates over all events received until timeout or event data is nil. You may set the timeout to 0 to effectively stop receiving events.

Parameters:

  • em emitter
  • timeout number (optional)

Usage:

    for ctx, event in pairs(emitter) do
   -- process event here
   if condition then ctx.timeout = 0 end --unsubscribe
end
await (em, f[, timeout])
Waits for the first event to match the predicate within the timeout.

Parameters:

  • em emitter
  • f function A predicate, which returns true if the function should return the event.
  • timeout number How long to to wait for events. (defaults to indefinitely.) (optional)

Returns:

    The event emitted.

Usage:

    local ntx = client.events.MESSAGE_CREATE:await(function(pending) return pending.user.id == client:me().id end)

objects

emitter
An emitter object. Set callbacks with listen and emit events with emit. All functions which take a emitter as their first argument can be called from the emitter in method form.

Fields:

  • pollfd Condition variable, signalled on emission.

Usage:

    E = emission.new()
E:listen(print)
E:emit"Hello, world!"
--> Hello, world!
E:remove(print)
E:emit"No one's listening :("
--> nothing happens
